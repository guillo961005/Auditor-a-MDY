Reglas esenciales de conexión

  • El flujo de secuencia no cruza límites de pool.
  • El evento de inicio no tiene flujos entrantes; el de fin, ni salientes.
  • Toda compuerta XOR etiqueta cada camino saliente.
  • El flujo de mensaje solo conecta pools distintos.
  • La asociación no sustituye al flujo de secuencia.

Buenas prácticas en MDY

  • Nombrar tareas como Verbo + Objeto (ej.: «Verificar presupuesto»).
  • Flujo de izquierda a derecha; minimizar cruces de líneas.
  • Usar pools solo si hay dos entidades autónomas distintas.
  • Anotar normativas aplicables (Res. 2674/2013, contratos) con anotaciones.
  • Objetos de datos para documentar entradas/salidas auditables.
